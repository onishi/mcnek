# 道の駅一覧 Web サービス計画

道の駅を探す、地図で見る、自分が行った駅を記録するための Web サービスを作る。

## 使うデータソース

まずは次の 2 種類を使う。

- 国土交通省 道の駅一覧
  - https://www.mlit.go.jp/road/Michi-no-Eki/list.html
  - 公式の登録情報として扱う
  - 駅名、登録回、登録年月、所在地の一覧を取得する
  - 2025-12-19 時点で 1,231 駅が登録されている
- 全国「道の駅」連絡会 道の駅検索
  - https://www.michi-no-eki.jp/stations/search/all/all/all
  - 詳細情報の補完元として扱う
  - 都道府県、市区町村、個別ページ、検索ページの情報を確認する

国土交通省の一覧をマスターデータにし、全国「道の駅」連絡会の情報を詳細情報として突き合わせる。両方に差分がある場合は、まず国土交通省の登録情報を正とし、差分を確認できる管理用レポートを出す。

## 最初に作る順番

### 1. データ取り込み基盤

最初に、画面よりも道の駅データを安定して扱える状態を作る。

- 国土交通省の XLS または PDF から一覧を取り込む
- 駅名、都道府県、市区町村、所在地、登録回、登録年月を正規化する
- 全国「道の駅」連絡会の検索結果から補完情報を取得する
- 取り込み結果を JSON または DB に保存する
- 重複、名称ゆれ、所在地不一致を検出する
- 取り込み処理のユニットテストを作る

この段階の完了条件:

- 全駅の一覧データをローカルで再生成できる
- 駅名、都道府県、所在地で検索できるだけの基本データがある
- 取り込み件数と差分を確認できる

### 2. 一覧ページ MVP

次に、サービスとして見える最小画面を作る。

- 道の駅を都道府県別に一覧表示する
- 駅名、都道府県、市区町村で検索できる
- 都道府県、地方、登録年月で絞り込める
- 各駅の詳細ページへ遷移できる
- スマートフォンでも使いやすい一覧 UI にする

この段階の完了条件:

- ユーザーが全国の道の駅を一覧で探せる
- 検索と絞り込みが実用できる
- 詳細ページで基本情報を確認できる

### 3. 詳細ページ

一覧から個別の道の駅を詳しく見られるようにする。

- 駅名、所在地、都道府県、市区町村、登録年月を表示する
- 公式サイトや連絡会ページへの外部リンクを表示する
- 地図表示に必要な緯度経度を持てるようにする
- 緯度経度が未取得の駅は所在地からジオコーディング候補を作る

この段階の完了条件:

- 各駅に固有 URL がある
- 最低限の詳細情報を安定して表示できる
- 地図実装に進めるデータ構造になっている

### 4. 地図ページ

一覧では見つけにくい近隣の道の駅を探せるようにする。

- 全国地図に道の駅をピン表示する
- 都道府県、地方、検索条件で地図上の表示を絞り込む
- ピンから詳細ページへ移動できる
- 現在地周辺の道の駅を探せる
- ピンの密集時はクラスタリングする

この段階の完了条件:

- 地図から道の駅を探せる
- 一覧検索と地図検索の条件が揃っている
- モバイルで地図操作が破綻しない

### 5. 自分が行ったリスト

ユーザーごとの記録機能を追加する。

- 道の駅を「行った」にできる
- 行った日、メモ、写真 URL などを保存できる
- 行った駅だけを一覧・地図で表示できる
- 都道府県別の訪問数を表示する
- ログインなしのローカル保存から始め、必要になったらアカウント同期を追加する

この段階の完了条件:

- ユーザーが訪問済みの道の駅を記録できる
- 一覧と地図で訪問済み状態がわかる
- データを消さずに将来のログイン機能へ移行できる

### 6. データ更新と運用

継続的に使えるサービスにする。

- データ取得スクリプトを定期実行できるようにする
- 新規登録、名称変更、所在地変更を検出する
- 変更差分をレビューしてから公開できるようにする
- データソースの取得失敗を検知する
- README または docs に更新手順を書く

この段階の完了条件:

- 最新の道の駅登録情報を追従できる
- 外部サイトの構造変更に気づける
- 手動確認が必要な差分を見落としにくい

## 優先しないこと

初期リリースでは、次の機能は後回しにする。

- 口コミ、評価、ランキング
- 写真アップロード
- 複数ユーザーのフォロー
- スタンプラリーの細かいルール対応
- 経路検索
- 営業時間や休業日の完全な自動更新

まずは「公式データを元に全国の道の駅を探せる」「地図で見られる」「自分が行った駅を記録できる」状態を優先する。

## 技術メモ

- TypeScript strict モードで実装する
- React は関数コンポーネントを基本にする
- 新規関数には Jest のユニットテストを追加する
- データ取り込み、正規化、差分検出は UI と分離する
- 地図ライブラリは実装時に選定する
- 緯度経度の取得元と利用条件は実装前に確認する
