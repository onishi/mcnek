# 道の駅一覧 Web サービス計画

道の駅を探す、地図で見る、自分が行った駅を記録するための Web サービスを作る。

## 使うデータソース

まずは次の 2 種類を使う。

- 国土交通省 道の駅一覧
  - https://www.mlit.go.jp/road/Michi-no-Eki/list.html
  - 公式の登録情報として扱う
  - 駅名、登録回、登録年月、所在地の一覧を取得する
  - 2025-12-19 時点で 1,231 駅が登録されている
- 全国「道の駅」連絡会 道の駅検索
  - https://www.michi-no-eki.jp/stations/search/all/all/all
  - 詳細情報の補完元として扱う
  - 都道府県、市区町村、個別ページ、検索ページの情報を確認する

国土交通省の一覧をマスターデータにし、全国「道の駅」連絡会の情報を詳細情報として突き合わせる。両方に差分がある場合は、まず国土交通省の登録情報を正とし、差分を確認できる管理用レポートを出す。

## ローカル起動手順

```sh
npm install
npm run dev
```

- ブラウザで `http://localhost:5173/` を開く
- `npm test`: ユニットテスト・コンポーネントテストを実行する
- `npm run lint`: ESLint でコードをチェックする
- `npm run format`: Prettier でコードを整形する
- `npm run format:check`: Prettier の整形チェックのみ行う（CI 向け）
- `npm run build`: 型チェックと本番ビルドを実行する

## データモデル

`src/types/roadsideStation.ts` で定義。

```ts
type Prefecture = "北海道" | "青森県" | ... ; // 47 都道府県のリテラル型
type Region =
  | "北海道地方"
  | "東北地方"
  | "関東地方"
  | "中部地方"
  | "近畿地方"
  | "中国地方"
  | "四国地方"
  | "九州・沖縄地方";

type RoadsideStation = {
  id: string;
  name: string;
  prefecture: Prefecture;
  municipality: string;
  address: string;
  registrationRound: number | null;
  registrationDate: string | null; // "YYYY-MM" 形式
  latitude: number | null;
  longitude: number | null;
  mlitSourceUrl: string;
  associationSourceUrl: string | null;
  updatedAt: string;
};

type VisitRecord = {
  stationId: string;
  visitedAt: string | null;
  memo: string | null;
  createdAt: string;
  updatedAt: string;
};
```

- 都道府県は 47 件のリテラル型 `Prefecture` で表す（`src/types/roadsideStation.ts`）
- 地方（`Region`）は `RoadsideStation` のフィールドとして持たず、`prefecture` から `getRegion()`（`src/data/regions.ts`）で導出する。データの重複・不整合を避けるため
- 登録年月は `"YYYY-MM"` 形式の文字列、または未登録時は `null`
- サンプルデータ 3 件は `src/data/sampleStations.ts` にある

## 優先しないこと

初期リリースでは、次の機能は後回しにする。

- 口コミ、評価、ランキング
- 写真アップロード
- 複数ユーザーのフォロー
- スタンプラリーの細かいルール対応
- 経路検索
- 営業時間や休業日の完全な自動更新

まずは「公式データを元に全国の道の駅を探せる」「地図で見られる」「自分が行った駅を記録できる」状態を優先する。

## 技術メモ

- TypeScript strict モードで実装する
- React は関数コンポーネントを基本にする
- 新規関数には Jest のユニットテストを追加する
- データ取り込み、正規化、差分検出は UI と分離する
- ルーティングは react-router-dom を使う（詳細ページの URL は `/stations/:id`）
- 地図ライブラリは実装時に選定する
- 緯度経度の取得元と利用条件は実装前に確認する
- 国土交通省データの XLS 取得 URL は `https://www.mlit.go.jp/road/Michi-no-Eki/file/list.xls`
- 国土交通省データの「所在地」列は市区町村までで、番地を含む完全な住所ではない。完全な住所はフェーズ 8（全国「道の駅」連絡会データ）で補完する想定
- XLS の取り込みは `npm run import:fetch` → `npm run import:mlit` の順に実行する（`data/raw/` はローカルキャッシュのため Git 管理外）
- 国土交通省データの「県名」「所在地」列は元から都道府県・市区町村に分かれているため、正規化では型の検証（`Prefecture` への絞り込み）が主な処理になる
- `id` は都道府県名と駅名から生成する安定したハッシュ値とする（連絡会データとの突き合わせはフェーズ 8 で別途行う）
- 緯度経度・連絡会の詳細 URL は国土交通省データのみでは取得できないため、正規化時点では `null` のままにする
